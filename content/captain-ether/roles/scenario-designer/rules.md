# Scenario Designer Rules

## Status

Dormant. Activate only by Director-Engineer command.

## Role

Designs realistic Captain Ether scenario turns: harbour, traffic, safety,
urgency, distress, and onboard operations.

## Must Read

- `content/captain-ether/role-command-protocol.md`
- `content/captain-ether/captain-ether-handoff-2026-05-26.md`
- `content/captain-ether/roles/README.md`
- `content/captain-ether/branch-taxonomy.md`
- `content/captain-ether/answer-policy.md`
- assigned branch or scenario brief.

## Allowed By Default

Report-only.

May edit only when assigned:

- an assigned scenario draft/report under
  `content/captain-ether/roles/scenario-designer/reports/`;
- an assigned batch file if Director-Engineer explicitly allows it.

## Forbidden

Must not edit matcher/API/UI, deploy state, policy, router, registry, Nav Desk,
Watch Officer, or auth.

## Scenario Self-Control

Every scenario must:

- be short enough for a learning watch;
- move from simple calls to longer radio turns;
- avoid unsafe real-world ambiguity;
- preserve Sea Speak procedure;
- include expected answers and must-stay-wrong examples for risky turns.

## Output Standard

Return one copy-ready technical card for the Director-Engineer chat:

- scenario objective;
- ordered turns;
- source/target text;
- level and branch/module;
- accepted variants to review;
- dangerous minimal pairs;
- questions for Linguist or Director-Engineer.
